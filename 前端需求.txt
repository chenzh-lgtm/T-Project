===========================================
微信心理诊断小程序 - 前端需求文档
===========================================

版本：v1.0
日期：2026年5月
作者：开发团队

---

一、项目概述
============
本小程序是一款心理诊断测试平台，提供多种性格测试和心理健康评估功能，帮助用户了解自身性格特点和心理状态。

---

二、功能需求
============

2.1 测试分类体系
----------------
小程序应支持以下测试分类：

2.1.1 性格测试类
    - MBTI性格测试（16型人格）
    - 九型人格测试
    - OCEAN大五人格测试
    - DISC行为风格测试
    - PDP性格测试
    - 艾森克人格问卷（EPQ）
    - 卡特尔16PF人格因素测试
    - 霍兰德职业兴趣测试
    - 荣格性格类型测试

2.1.2 心理健康测试类
    - SCL-90症状自评量表
    - MMPI-2明尼苏达多项人格测验
    - PHQ-9患者健康问卷（抑郁筛查）
    - SDS抑郁自评量表
    - DASS-21抑郁-焦虑-压力量表
    - SAS焦虑自评量表
    - 贝克抑郁量表（BDI）
    - 汉密尔顿抑郁量表（HAMD）
    - 汉密尔顿焦虑量表（HAMA）

2.1.3 趣味测试类
    - 星座性格测试
    - 职业倾向测试
    - 人际关系测试

2.2 核心功能流程
----------------

2.2.1 测试选择页面
    - 展示测试分类导航（性格测试、心理测试、趣味测试）
    - 每个分类下列出可用的测试项目
    - 显示测试简介、题目数量、预计完成时间
    - 支持搜索和筛选功能

2.2.2 测试答题页面
    - 按顺序展示题目
    - 支持单选题、多选题、量表题（1-5分评分）
    - 显示当前进度（第X题/共Y题）
    - 支持返回上一题修改答案
    - 支持中途保存退出，下次继续

2.2.3 结果分析页面
    - 展示测试结果概述
    - 详细分析各项指标得分
    - 提供专业解读建议
    - 支持分享测试结果
    - 支持生成测试报告图片

2.3 用户系统
------------
    - 微信授权登录
    - 测试历史记录
    - 收藏喜欢的测试
    - 个人测试报告管理

---

三、页面结构
============

3.1 首页（首页）
    - 顶部：小程序名称、用户头像
    - 中部：测试分类入口（图标+名称）
    - 底部：推荐测试列表

3.2 测试列表页
    - 分类标签切换
    - 测试卡片列表
    - 搜索框

3.3 测试详情页
    - 测试介绍
    - 开始测试按钮
    - 测试说明

3.4 答题页面
    - 题目内容展示
    - 选项选择区域
    - 进度条
    - 上一题/下一题按钮

3.5 结果页面
    - 结果标题和得分
    - 详细分析内容
    - 建议与指导
    - 分享和保存按钮

3.6 个人中心页
    - 用户信息
    - 测试历史
    - 收藏列表
    - 设置

---

四、交互设计
============

4.1 导航交互
    - 底部Tab栏：首页、分类、个人中心
    - 页面间平滑过渡动画

4.2 答题交互
    - 选项点击反馈（选中状态高亮）
    - 进度实时更新
    - 提交时确认弹窗

4.3 结果交互
    - 结果展示动画效果
    - 分享功能（微信好友、朋友圈）
    - 保存报告到相册

---

五、UI设计规范
==============

5.1 配色方案
    - 主色调：温暖渐变（蓝紫色系）
    - 辅助色：柔和的绿色、橙色用于强调
    - 背景：浅色渐变，舒适护眼

5.2 字体规范
    - 标题：较大字号，加粗
    - 正文：清晰易读的字号
    - 提示文字：较小字号，浅灰色

5.3 图标风格
    - 简洁现代的线性图标
    - 统一的视觉风格

---

六、技术要求
============

6.1 小程序基础要求
    - 兼容微信小程序最新版本
    - 支持iOS和Android双平台
    - 响应式适配不同屏幕尺寸

6.2 性能要求
    - 页面加载时间 < 2秒
    - 答题流畅无卡顿
    - 结果计算实时响应

---

七、非功能性需求
================
7.1 数据隐私
    - 用户测试数据加密存储
    - 遵守隐私保护法规

7.2 用户体验
    - 界面简洁美观
    - 操作流程清晰
    - 提供友好的错误提示

---

八、前端技术规范
================
8.1 技术栈
----------
    - 框架：微信小程序原生框架（WXML + WXSS + JavaScript）
    - 开发工具：微信开发者工具
    - 语言：JavaScript ES6+
    - 样式：WXSS（支持rpx响应式单位）

8.2 目录结构
------------
    ├── app.js              # 小程序入口文件
    ├── app.json            # 全局配置
    ├── app.wxss            # 全局样式
    ├── pages/              # 页面目录
    │   ├── index/          # 首页
    │   ├── test-list/      # 测试列表页
    │   ├── test-detail/    # 测试详情页
    │   ├── test/           # 答题页面
    │   ├── result/         # 结果页面
    │   └── profile/        # 个人中心页
    ├── components/         # 公共组件
    │   ├── test-card/      # 测试卡片组件
    │   ├── question-item/  # 题目组件
    │   └── progress-bar/   # 进度条组件
    ├── utils/              # 工具函数
    │   ├── api.js          # API封装
    │   ├── storage.js      # 本地存储工具
    │   └── utils.js        # 通用工具函数
    ├── data/               # 模拟数据
    └── images/             # 静态图片资源

8.3 命名规范
------------
    - 文件命名：小写字母 + 连字符（如 test-list）
    - 组件命名：大驼峰命名（如 TestCard）
    - 变量命名：小驼峰命名（如 testList）
    - 常量命名：全大写 + 下划线（如 API_BASE_URL）

8.4 开发规范
------------
    - 页面生命周期钩子必须完整实现
    - 事件处理函数统一以 on 开头（如 onStartTest）
    - 异步操作必须处理 loading 和错误状态
    - 代码注释清晰，关键逻辑必须有注释

8.5 API调用规范
---------------
    - 统一封装在 utils/api.js 中
    - 使用 Promise 方式调用
    - 请求前显示 loading，响应后隐藏
    - 统一处理错误状态码
    - 支持请求超时设置

8.6 安全规范
------------
    - 所有用户输入必须进行校验
    - 敏感数据传输使用 HTTPS
    - 本地存储数据加密处理
    - 防止 XSS 攻击（对用户输入进行转义）
